# Fix Player Focus Flow — implementation + verification

## What changed

`e2e/flows/player-focus.mjs` — replaced the page-wide `.or()` Play/Resume
button locator with a per-card-scoped `nth(index)` iteration, mirroring
`e2e/flows/player-uat.mjs` lines ~85-108 exactly:

```js
// before
const playButton = page
  .locator('.media-card button', { hasText: /^Play$/ })
  .first()
  .or(page.locator('.media-card button', { hasText: /^Resume$/ }).first());

// after
const cardCount = await page.locator('.media-card').count();
let playButton = null;
for (let index = 0; index < cardCount; index += 1) {
  const card = page.locator('.media-card').nth(index);
  const button = card.locator('button', { hasText: /^(Play|Resume)$/ }).first();
  if (await button.count().then((n) => n > 0)) {
    playButton = button;
    break;
  }
}
if (!playButton) {
  await shootView(page, `${flowName}-03-no-play-button`);
  fail('player-focus: found no Play/Resume button to open the modal');
}
```

No other line in the flow was touched. Every assertion after the button is
located and clicked — dialog receives focus (not the opener), `overflow:
hidden` scroll lock, iframe mounts at the same-origin `/player` route, Space
is a no-op (dialog count stays 1, focus stays on dialog, scroll unchanged),
Escape closes the dialog and restores scroll — is byte-for-byte identical to
before.

Only `e2e/flows/player-focus.mjs` was edited. `player-uat.mjs`, `run.mjs`,
sandbox provisioning, and all other flows are untouched.
