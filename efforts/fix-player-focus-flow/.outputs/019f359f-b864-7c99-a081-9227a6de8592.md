### Environment setup

This worktree (`fix-player-focus-flow.1783311088526`) had no sandbox
provisioned yet (`.env.sbx` / `config.sbx.json` missing, fresh volumes). Ran
the standard one-time bring-up: `up -d jellyfin-sandbox`, `run --rm
sandbox-generate`, `run --rm sandbox-provision`, then `up -d` for the full
stack, then `run --rm sandbox-reset` for a guaranteed clean played-state
baseline before the first proof run.

Port 8080 was already bound by another worktree's `gogglebox-proxy-1`
(different Compose project). `player-focus`/`mark-all-watched` don't depend
on the host-published proxy port (Playwright drives `http://client:5173`
inside the docker network), so a temporary untracked
`docker-compose.local-port-override.yml` (proxy republished on host 8081)
was layered on just to let `docker compose run` reconcile without erroring
on the port conflict. Deleted immediately after all proof runs completed;
`git status --short` confirms no leftover tracked or untracked files besides
the intended `.proofs/` additions.

Also discovered: `PROOF_RUN_ID` must be passed via `-e PROOF_RUN_ID=...` on
every `proof` invocation (a bare shell-exported `PROOF_RUN_ID` does not reach
the container — compose `environment:` only sees `-e` flags on `run`,
matching the existing `PROOF_URL` gotcha documented in the KB). Without it,
the first run landed at `./artifacts/<timestamp>/` instead of
`./artifacts/<PROOF_RUN_ID>/<timestamp-flow>/`; corrected by re-running with
`-e PROOF_RUN_ID=...` explicitly on every invocation from then on.
