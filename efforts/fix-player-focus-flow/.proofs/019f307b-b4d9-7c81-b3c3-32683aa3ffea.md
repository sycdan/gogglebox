# AC2 proof — passes in both single-card and multi-card sandbox states

Produced directly by gogglebox-prover against a freshly provisioned sandbox
stack in worktree `fix-player-focus-flow.1783311088526` (sandbox Jellyfin was
generated/provisioned from scratch in this worktree: `up -d jellyfin-sandbox`,
`run --rm sandbox-generate`, `run --rm sandbox-provision`, then
`run --rm sandbox-reset` for a guaranteed clean played-state baseline before
the first run). All screenshots cited below were copied from
`./artifacts/019f35a3-9479-70f2-858a-a6e5f593553e/...` into this `.proofs/`
directory immediately after the run, so they are durable and survive worktree
teardown (unlike the prior, rejected version of this proof doc, which cited a
gitignored `./artifacts/...` path directly).

Port 8080 was already bound by another worktree's long-running
`gogglebox-proxy-1` container (a different Docker Compose project). A
temporary untracked `docker-compose.local-port-override.yml` (republishing
`proxy` on host `8081` via `ports: !override`) was layered on for this
session's own runs only, and was deleted immediately after proving — `git
status --short` confirms no tracked or leftover untracked files remain.
`player-focus` itself never depends on the host-published port — it drives
Playwright against `http://client:5173` inside the docker network — so this
override only avoided a host-port collision on the `proxy` service, nothing
else.

Shared `PROOF_RUN_ID=019f35a3-9479-70f2-858a-a6e5f593553e` was passed via
`-e PROOF_RUN_ID=... -e PROOF_FLOW=...` on every `proof` invocation (a bare
shell-exported `PROOF_RUN_ID` does NOT reach the container — compose
`environment:` only picks up variables passed with `-e` on `run`), so all
screenshots from every invocation below are grouped under one batch dir.
