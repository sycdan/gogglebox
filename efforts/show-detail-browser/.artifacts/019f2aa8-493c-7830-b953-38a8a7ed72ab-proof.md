## AC3 — Each visible episode row shows watched/unwatched state for every watcher in the active group

### Status: PASS (ui-tested)

### Stack / data

Same run as AC1/AC2: `./scripts/sbx.sh` sandbox stack,
`e2e/flows/show-detail-browser.mjs` via `PROOF_FLOW=show-detail-browser`,
party = all 4 sandbox viewers (Alice, Bob, Carol, Dave), show = seeded
fixture **"Normal Show"**.

To make the per-viewer evidence unambiguous (not just "everyone unwatched"),
the flow first resets all played-state (`resetAllPlayedState`) and then seeds
a **mixed** state directly via the sandbox Jellyfin API: marks **Alice** as
having watched S01E01 "Pilot", leaving Bob/Carol/Dave unwatched for that same
episode.

### What was driven

1. Opened the "Normal Show" detail modal (all seasons).
2. On the first visible episode row (S01E01 "Pilot"), located the
   `.viewer-pills` group and confirmed 4 `.viewer-pill` elements — one per
   active-party viewer (Alice, Bob, Carol, Dave).
3. Read each pill's `title` attribute and `watched` class/check-badge:
   - Alice — `watched=true`, green `.viewer-pill-check` badge present.
   - Bob — `watched=false`, no check badge.
   - Carol — `watched=false`, no check badge.
   - Dave — `watched=false`, no check badge.
4. Confirmed the pills are **display-only**: clicked Alice's pill and
   confirmed its watched state did NOT change afterward (`true` before,
   `true` after) — this modal does not add a watched-state editing surface,
   respecting the effort's nongoal.
