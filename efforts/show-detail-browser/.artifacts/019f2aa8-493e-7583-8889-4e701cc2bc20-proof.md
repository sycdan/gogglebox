## AC4 — In-modal Jellyfin search finds episodes by keyword, scoped to one show only

### Status: PASS (ui-tested)

### Stack / data

Same run as AC1/AC2/AC3: `./scripts/sbx.sh` sandbox stack,
`e2e/flows/show-detail-browser.mjs` via `PROOF_FLOW=show-detail-browser`,
party = all 4 sandbox viewers, show = seeded fixture **"Normal Show"**. The
keyword `"Pilot"` matches exactly one episode (S01E01) of "Normal Show" and no
episode of any other seeded show.

### What was driven

1. Opened the "Normal Show" detail modal and confirmed the in-modal search
   field's label reads **"Search episodes in Normal Show"** — explicitly
   scoped by name, distinct from the top-level toolbar search.
2. Before typing into the modal search, snapshotted the **top-level toolbar
   search box's value** (`"Normal Show"`, left over from locating the card
   earlier in the flow) and its **"Search results" rail's titles**
   (`["Normal Show"]`).
3. Typed `"Pilot"` into the in-modal search field and waited for the
   debounced request.
4. Confirmed exactly 1 episode row remained, titled `"Pilot"` — no other
   episodes of "Normal Show", and (by construction, since the query is
   scoped server-side via `ParentId=<seriesId>&SearchTerm=Pilot`) no episodes
   of any other show.
5. Re-snapshotted the top-level toolbar search box value and its results
   rail titles **after** the in-modal search: both **byte-for-byte
   identical** to the "before" snapshot (`"Normal Show"` /
   `["Normal Show"]`) — the in-modal search did not touch, clear, or extend
   the separate global search state in any way.

### Judgement

Proved. The in-modal search is a separate, explicitly-labelled, per-show
scoped control (`GET /api/shows/:seriesId/episodes?q=Pilot`, which always
carries `ParentId=<seriesId>` per the builder's implementation) — it finds
the keyword-matching episode within the open show only, and the top-level
toolbar/global search box and its results rail are provably unaffected
(identical value and result set before vs. after), so this cannot be
mistaken for — or regress into — a global/discovery-rail search, per the
effort's nongoal.
