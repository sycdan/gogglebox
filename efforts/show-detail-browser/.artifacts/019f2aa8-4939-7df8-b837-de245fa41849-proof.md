## AC1 — Clicking a show title opens an accessible show modal without breaking page state

### Status: PASS (ui-tested)

### Stack / data

- `./scripts/sbx.sh` — seeded, offline sandbox Jellyfin (deterministic fixtures
  from `tools/sandbox/fixtures.mjs`), `server` + `client` + `proxy` up via
  `./scripts/sbx.sh up -d`.
- Proof flow: `e2e/flows/show-detail-browser.mjs` (new module, wired into
  `e2e/run.mjs`), driven with
  `./scripts/sbx.sh run --rm -e PROOF_RUN_ID=<id> -e PROOF_FLOW=show-detail-browser proof`.
- Party: all 4 sandbox viewers (Alice, Bob, Carol, Dave).
- Show under test: seeded fixture **"Normal Show"** (2 seasons: S1×4, S2×3 —
  `tools/sandbox/fixtures.mjs` `SHOWS[0]`).

### What was driven

1. Located a media-card title button for "Normal Show" (via the toolbar
   search, since it wasn't on the initial recommendations rail for this
   party) and captured the section headings on the page
   (`["Continue watching","Search results"]`) **before** clicking it.
2. Clicked the title button (`button.link-title` wrapping the card's `<h3>`).
3. Confirmed a `.modal.show-detail-modal` became visible with
   `role="dialog"`, `aria-modal="true"`, and a descriptive
   `aria-label="Normal Show episodes"` — accessible dialog semantics.
4. Confirmed focus moved into the modal (`document.activeElement` was the
   modal `<div>` itself, which is focusable and receives keyboard focus per
   the existing player-modal focus pattern).
5. Pressed `Escape` and confirmed the modal became hidden.
6. Re-captured the section headings **after** closing the modal — identical
   list (`["Continue watching","Search results"]`), proving opening/closing
   the modal is a purely local state change that never touched
   recommendations/continue-watching/search state behind it.
7. Separately, opened the modal a second way: clicking the series-name link
   (`button.link-title-inline`) inside a **continue-watching** card's `.meta`
   line (found "Near Finale" on this party's rail) — confirmed the same
   `.modal.show-detail-modal` opened for that show too, proving "clicking a
   show title anywhere it appears" covers both media-card and
   continue-watching entry points.

### Judgement

Proved. Both click-sites (media-card title, continue-watching series-name
link) open an accessible `role="dialog"` modal with a descriptive
`aria-label`, and the page's section headings are byte-identical before vs.
after open/close — no mutation of the page state behind the modal.
