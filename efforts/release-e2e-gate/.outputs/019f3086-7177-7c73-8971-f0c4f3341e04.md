## AC3 — new `e2e` job

`if: needs.gate.outputs.bump == 'true'` guard as `test`). Steps, in order:

1. `actions/checkout@v4`
2. `docker compose -f docker-compose.yml -f docker-compose.sbx.yml up -d jellyfin-sandbox`
3. `... run --rm sandbox-generate`
4. `... run --rm sandbox-provision` (mints `.env.sbx` + `config.sbx.json` at repo root)
5. `... run --rm sandbox-reset`
6. `... up -d` (full stack: jellyfin-sandbox + server + client + proxy) — only
   after step 4 has produced the generated env/config the sbx overlay's
   `server`/`proof` overrides require.
7. Poll loop (`for i in $(seq 1 60); do ...; sleep 5; done`) reading
   `docker compose ... ps --format json client | jq -r '.Health // .State'`
   until it equals `"healthy"`, else fails after ~5 minutes and dumps
   `docker compose ... logs client` for diagnosis. No fixed sleep as the sole
   wait strategy.
8. `... run --rm -e PROOF_URL=http://proxy:8080 -e PROOF_FLOW=all proof` — the
   full flow suite in one invocation via the all-flows mode landed in AC1/AC2,
   through the same-origin proxy target `player-handoff` requires. A non-zero
   exit here fails the step/job the normal way a CI `run:` step does.
