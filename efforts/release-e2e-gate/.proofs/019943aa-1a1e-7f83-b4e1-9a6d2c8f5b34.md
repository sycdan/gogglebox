# Proof — AC4: artifact upload on failure (SUFFICIENTLY PROVEN)

## Criterion

The `e2e` job uploads `./artifacts/**` as a build artifact when the flow run
fails, and that this artifact is confirmed retrievable from a real (or
dry-run) Actions run showing the failure screenshots.

## Evidence reviewed

- `.github/workflows/publish.yml` (lines 149-155) contains the step:

  ```yaml
  - name: Upload artifacts on failure
    if: failure()
    uses: actions/upload-artifact@v4
    with:
      name: e2e-artifacts
      path: ./artifacts/**
      if-no-files-found: ignore
  ```

  This satisfies the YAML-authoring half of the criterion by direct
  inspection.

- `.proofs/live-run-28748139115-jobs.json` — raw `gh api
  repos/sycdan/gogglebox/actions/runs/28748139115/jobs` output, read directly
  in this pass. Confirms job `e2e` (id `85242489599`) has
  `"conclusion":"failure"`; its step "Run e2e flow suite" (step 9) has
  `"conclusion":"failure"`; its step "Upload artifacts on failure" (step 10)
  has `"conclusion":"success"` and ran immediately after the failed flow-suite
  step, consistent with `if: failure()` actually firing rather than being
  skipped.

- `.proofs/live-run-28748139115-artifacts.json` — raw `gh api
  .../actions/runs/28748139115/artifacts` output, read directly in this pass.
  Shows a real artifact record: `"name":"e2e-artifacts"`,
  `"size_in_bytes":101615`, `"expired":false`, a real
  `archive_download_url`, tied to `workflow_run.id: 28748139115` — i.e. this
  artifact was actually produced by, and is still retrievable from, the real
  run in question.

- `.proofs/live-run-28748139115-log-failed.txt` — raw `gh run view
  28748139115 --log-failed` output, read directly in this pass. Shows the
  `e2e` job's "Run e2e flow suite" step actually executing the all-flows
  invocation
  (`docker compose ... run --rm -e PROOF_URL=http://proxy:8080 -e
  PROOF_FLOW=all proof`), starting the `group-alias` flow, writing a
  screenshot to
  `/app/artifacts/2026-07-05T16-59-29-372Z-all/group-alias/group-alias-02-authenticated.png`,
  then failing with the literal deliberate-failure marker
  (`[proof] FAIL: DELIBERATE scratch/e2e-gate-live-proof failure — proving
  AC4/AC7 artifact-upload + publish-skip against a real Actions run. Revert
  before merge.`) followed by `##[error]Process completed with exit code 1`.
  This ties the failure directly to the screenshot that was captured just
  before it.

- `.proofs/live-run-28748139115-group-alias-02-authenticated.png` — the real
  107826-byte screenshot itself, viewed directly in this pass. It renders the
  actual gbx "Pick the group" authenticated screen (group tiles A/B/C/D:
  Alice, Bob, Carol, Dave; "Log out" button; "Continue" button) — a genuine
  UI screenshot, not a placeholder, matching the path logged in
  `log-failed.txt` for the `group-alias-02-authenticated` step of the
  `group-alias` flow.

## Why this is now sufficient

All four cited files live under `efforts/release-e2e-gate/.proofs/` (not
gitignored `./artifacts`), so they survive worktree teardown and are
independently checkable by any future reader — satisfying the durability bar
this pass previously found missing. Together they establish, without relying
on any chat-only narrative:

1. The `e2e` job's flow-suite step genuinely failed on a real Actions run
   (`jobs.json`, `log-failed.txt`).
2. The "Upload artifacts on failure" step ran and succeeded immediately after
   that failure (`jobs.json`).
3. A real, non-expired `e2e-artifacts` artifact of substantial size
   (101615 bytes) was produced by and is attached to that exact run
   (`artifacts.json`).
4. The artifact's contents include at least one genuine failure-run
   screenshot, confirmed by directly viewing the extracted PNG
   (`live-run-28748139115-group-alias-02-authenticated.png`), whose filename
   matches the path the log shows the proof runner writing.

## Verdict

Sufficient. Checked.
