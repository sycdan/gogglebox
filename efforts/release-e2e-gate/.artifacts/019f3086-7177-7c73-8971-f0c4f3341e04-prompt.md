# Phase: implement AC3-7 of release-e2e-gate

Read `efforts/release-e2e-gate/ReleaseE2eGate.md` in full for context
(Overview, Goals, Nongoals). AC1 and AC2 are already done and landed on
main — `e2e/run.mjs` now supports `PROOF_FLOW=all` (all-flows mode, isolated

This handoff covers the remaining five acceptance criteria, all in
`.github/workflows/publish.yml`:

**AC3** — Add a new `e2e` job, parallel to the existing `test` job (both
`needs: gate`), that:
- bootstraps the sandbox from a clean runner by running the existing sandbox
  commands in order: bring up `jellyfin-sandbox`, run `sandbox-generate`, run
  `sandbox-provision` (creates `.env.sbx` and `config.sbx.json`), then run
  `sandbox-reset`,
- brings up the full sandbox stack (`docker-compose.yml` +
  `docker-compose.sbx.yml`, equivalent to `./scripts/sbx.sh up -d`) only
  after that generated sandbox config exists,
- waits for the client's existing healthcheck to report healthy before
  proceeding (no fixed sleep as the sole wait strategy — poll/retry against
  the healthcheck, e.g. via `docker compose ps` health status or an
  equivalent condition loop),
- runs the full flow suite in one invocation via the new all-flows mode
  through the same-origin proxy target required by `player-handoff`
  (`-e PROOF_URL=http://proxy:8080 -e PROOF_FLOW=all`), and fails the job if
  that process exits non-zero.

**AC4** — On failure, upload `./artifacts/**` as a build artifact (e.g.
`actions/upload-artifact@v4` with `if: failure()`) so screenshots from a
broken CI run are inspectable from the Actions UI. Confirm — via a real or
dry-run Actions run showing a deliberate failure — that the artifact is
actually retrievable, not just that the YAML looks right.

**AC5** — Tear down the sandbox stack after the job finishes regardless of
pass/fail (e.g. `if: always()` step running the equivalent of
`docker compose down -v --remove-orphans` for both the base and sbx overlay).
Confirm no dangling containers are left on the runner after a run.

**AC6** — Run the `e2e` job under `timeout-minutes: 60`, with a workflow
comment documenting this as an initial tunable value (covers sandbox
bootstrap plus 14 sequential Playwright flows against sandbox Jellyfin), not
a permanent constant.

**AC7** — Extend `publish`'s `needs:` to include both `test` and `e2e`, so
neither a unit-test failure nor an e2e-flow failure can result in a published
image. Prove this with a deliberately-broken flow (temporary local/dry-run
repro, reverted after proving) causing `publish` to be skipped exactly as a
deliberately-broken unit test does today. Confirm `release.yml`'s promotion
behavior (already out of scope/unchanged) is untouched — do not modify
`release.yml` at all.
