# Judgement Day

## Overview

Reimagine Gogglebox as a focused household video discovery experience. Instead of a passive
continue-watching portal with search bolted on, this effort makes Continue Watching one useful
recommendation channel in a single responsive discovery rail. The rail is backed by deterministic
service APIs and simple controls for choosing channels, rerolling suggestions, and filtering the
same discovery result set when the household already knows what it wants.



## Goals

- Define the recommendation inputs, ranking signals, and output contract.
- Keep recommendations explainable enough for users to trust the suggestions.
- Allow the service to evolve without coupling the UI to one ranking strategy.
- Provide support for an arbitrary number of recommendation channels.
- Begin thinking of "continue watching" as just the first recommendation channel.
- Add bounded controls for channel selection, rerolling, and search that filters the single rail.
- Constrain each rail page by responsive layout capacity, with mobile phones showing one item per
  page in a Tinder-for-TV style.

## Nongoals

- Do not call paid external AI or recommendation APIs unless a later effort
  explicitly approves that dependency.
- Do not introduce a separate recommendation mode or modal for search results.
- Do not replace non-search browse workflows such as show-detail browsing.
