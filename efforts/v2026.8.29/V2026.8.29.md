# Judgement Day

## Overview

Gogglebox optimizes **time-to-play for a room**, not engagement. The north-star
interaction: log in → "Press Start" → the top recommendation plays. The
north-star metrics are time-to-play (login → playback start) and
deck-advance/new-deck counts per session, both recorded as facts. The design is
deliberately anti-engagement: no autoplay dark patterns, no infinite browse.
Every layer degrades gracefully to a zero-config default — a fresh deploy with
no answered questions still gets a good rail. Explainability is the trust
engine: never surface an item without a renderable reason.

The architecture is a pipeline: **facts → signals → aggregation →
deterministic score → deck**.

1. **Fact store**: an append-only JSONL event log in the state dir
   (`{ ts, userId, itemId, kind, value, provenance }`). New data source = new
   fact kind, no schema migration. Provenance distinguishes `own-pad`
   (self-reported via a bound controller) from `room-reported`.
2. **Signal plugins**: each signal scores an item 0..1 with an optional reason,
   or abstains (`null`); weights renormalize over non-abstaining signals so
   missing data never punishes an item. Adding a signal = one module + one
   registry entry + one config weight.
3. **Party aggregation**: each signal declares its aggregator across present
   members — least-misery (min) for veto-shaped signals, mean for taste-shaped
   ones. Negative facts weigh roughly 2-3x positives: group satisfaction tracks
   the least-happy member.
4. **Hard filters** (pre-score): kind, genre, kids-only, hard time-budget
   cutoff; time budget also reappears as a soft fit signal.
5. **Deck**: score all candidates deterministically → top-K (~20) →
   score-weighted shuffle with a per-session seed → deal "tonight's deck"
   (~7). The deck is finite, shows position ("3 of 7"), and is reload-stable
   under the same session seed. Exhausting it offers an explicit "deal new
   deck" (fresh sample excluding seen; logged as a strong dissatisfaction
   fact). Deck advancement replaces the old reroll concept entirely.
6. **Explanation**: each shown item renders its top ~2 contributing reasons
   (by weight × value) as human sentences.

There is no long-running ranking daemon: scoring is on-demand (household scale
is milliseconds); only expensive Jellyfin aggregates may be cached. Rewatch
must be possible — the watched-union exclusion must not make comfort rewatches
unrecommendable, since a watched-by-all favorite is the group's highest-floor
pick. Eval discipline: an offline replay harness (hold out each party's last
watch, measure hit-rate@10 over the fact history; also track Start-press
acceptance) runs deterministically against the sandbox, and no new signal
lands without a named failure it fixes and an eval delta showing it.

The surface is a **proposal deck** (hero card + on-deck strip), not a
capacity-paged rail, driven by a **controller-first spatial focus** input
model.

## Goals

- Make time-to-play for the room the north-star metric, recorded as facts
  alongside deck-advance and new-deck counts.
- Deliver fact-driven, explainable recommendations: every surfaced item
  carries renderable reasons.
- Aggregate per-member signals into a party score with least-misery semantics
  for veto-shaped signals and heavier negative weighting.
- Replace paging and reroll with the deterministic seeded deck model (finite
  deck, position indicator, explicit deal-new-deck).
- Ship a controller-first input model: one spatial focus engine, keyboard and
  gamepad adapters, Press Start plays the hero.
- Preserve a zero-config floor: a fresh deploy with no facts and no answered
  questions still gets a good, explainable rail.

## Nongoals

A wont-do-yet ledger:

- **watch-plan** (score-maximal multi-episode sequences under a time budget —
  a knapsack over ranked items): deferred; the design is intentionally
  recorded here so it can be picked up later.
- **channel-preferences UI**: dropped — channels dissolve into candidate
  sources under one ranker; per-item correction beats channel toggles; a
  channel control panel violates the zero-config floor. Provenance survives as
  card explanations, not a control panel.
- **swipe/Tinder-style gesture protocol**: dropped as unhelpful.
- **multi-pad quorum/voting mechanics and phone-as-controller**: v1.5+ only;
  claim-to-join party formation and parallel answers are specced but ordered
  after core v1.
- **database-backed persistence**: JSONL facts until measured pain
  (supersedes [persistence-refactor](../persistence-refactor/PersistenceRefactor.md)).
- **paid external AI/recommendation APIs**: unchanged nongoal.
- **replacing browse workflows (show-detail)**: unchanged nongoal.
