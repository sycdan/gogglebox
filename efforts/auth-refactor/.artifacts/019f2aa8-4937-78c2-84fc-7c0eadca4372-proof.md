## How proved (UAT — real Jellyfin)

Ran `PROOF_FLOW=player-uat` against the real Jellyfin at `.env.uat` /
`config.uat.json` (`JELLYFIN_URL=http://htpc.lan:8096`, real account/library,
one primary user "Alice" preselected), via
`docker compose -f docker-compose.yml -f docker-compose.uat.yml`, run
explicitly with `-e PROOF_URL=http://proxy:8080` (required for the same-origin
`/player` handoff) and `--no-deps` for the `proof` container (server/client/
proxy were brought up fresh first via `up -d`/`up -d --force-recreate` in this
session's own compose project; the host's proxy port 8080 was already
occupied by an unrelated, already-running stack elsewhere on this machine —
worked around the same way as the prior pass, with a local, non-committed
`docker-compose.local.yml` clearing this project's proxy host-port publish,
which does not change the app's own configuration).

`htpc.lan` was confirmed reachable from this environment (`ping` succeeded)
and the server logs show it derived config from the real instance
(`[startup] derived effective config (schemaVersion 2, ...)`).

This flow was written by `gogglebox-builder` specifically to be UAT-safe,
unlike the two flows the prior pass tried and found were not:

- `group-alias` hard-codes the sandbox's two-primary fixture (Alice + Bob),
  which doesn't hold against the real UAT config (one primary only).
- `player-focus` uses a page-wide Play/Resume button locator that hits a
  Playwright strict-mode violation once several real-library cards each have
  their own Play/Resume button.

`player-uat` instead (a) selects whatever primaries are already preselected
with no fixed name/count assumption, falling back to the first plain viewer
card if none are preselected, and (b) scopes the Play/Resume search to ONE
`.media-card` at a time, picking the first card that has a Play or Resume
button, rather than a page-wide locator.

Result against real UAT data: **PASS**, exit 0 (`[proof] OK`).
`[proof] player-uat: keeping 1 preselected primary card(s) as-is` (Alice, the
real single-primary shape) into the main app, then
`chosen card = "(500) Days of Summer"`, the player modal opened
(`player-uat: PASS — player modal opened ... against real library data`), and
the Jellyfin `/player` child frame settled logged-in:
`jellyfin-web state = {"loggedInVisible":true,"loginFormVisible":false,
"hash":"#/details?id=...&play=true&autoplay=true&resume=true&...
serverId=..."}` — a real resume-in-progress item with a real serverId, not a
sandbox fixture.
