# AC2 — HTTP-route-level alias parity test

## What was checked

Read `src/server/server.test.ts` in full (316 lines) and the `createApp()`
extraction it depends on in `src/server/server.ts`.

Confirmed the test file drives a real `node:http` listener (not a mocked
Express app): `startTestServer()` calls `http.createServer(app).listen(0,
'127.0.0.1', ...)` against an app built by `createApp(config, jellyfin,
appState)`, and every request goes over the wire via global `fetch` against
`http://127.0.0.1:<ephemeral-port>`.

Confirmed all five required alias pairs are exercised, each asserting
`assert.deepEqual` between the group-prefixed and party-prefixed response
bodies:

1. `POST /api/party` vs `POST /api/group` (create) — bodies deep-equal,
   `{ ok: true, activeViewerIds: [...] }`.
2. `POST /api/party/verify-pins` vs `POST /api/group/verify-pins` — bodies
   deep-equal, `{ ok: true }`.
3. `GET /api/parties` vs `GET /api/groups` — bodies deep-equal; additionally
   asserts `partiesBody.parties[0].partyKey === partiesBody.groups[0].groupKey`
   (per-entry field-name-aliased equality), matching the AC's explicit
   requirement.
4. `POST /api/party/clear` vs `POST /api/group/clear` — bodies deep-equal,
   `{ ok: true, activeViewerIds: [] }`.
5. `GET /api/session` — asserts `sessionBody.activeGroupAlias ===
   sessionBody.activePartyAlias` (both equal to the seeded alias `'Alice +
   Bob'`).

Each test authenticates first via a real `POST /api/auth/login` call against
`buildConfig()`'s single `test-token` -> `household` account, carrying the
session cookie manually between requests (`cookieHeaderFrom`), matching real
client behavior rather than bypassing auth.

No live Jellyfin call is exercised: every test pre-seeds
`appState.setPartyPlayerUser(...)` for the fixed Alice/Bob member set before
starting the server, so `handleCreateParty`'s only Jellyfin-calling branch
(`!exists` -> `jellyfin.ensurePartyUser`) is never taken. `JELLYFIN_URL` /
`JELLYFIN_API_KEY` are stubbed with dummy unreachable values in
`test.before()` purely to satisfy `loadConfig()`'s import-time validation, not
because any test call reaches Jellyfin.

## Approach confirmation

Matches the Goals/Nongoals: plain `node:test` + `node:http` + global `fetch`
against an ephemeral listener, no supertest or other HTTP framework added.
Consistent with the existing `src/server/*.test.ts` convention (plain
`node:test`/`node:assert/strict`, e.g. `appState.test.ts`,
`parties.test.ts`).
