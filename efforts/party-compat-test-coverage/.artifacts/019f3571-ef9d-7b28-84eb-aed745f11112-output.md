  values match pre-computed hex strings`, asserting `derivePartyKey` for three

## AC2 — HTTP-route-level test for /api/group* aliases

Satisfied. Approach chosen: plain `node:test` + `node:http` + global `fetch`
against an ephemeral `app.listen(0)`-equivalent
(`http.createServer(app).listen(0, '127.0.0.1')`), per the Goals/Nongoals
preference — no supertest or other HTTP test framework was needed or added.

A minimal additive seam (`createApp()`) was required, exactly as the
Nongoals anticipated: `server.ts`'s module body unconditionally (a) calls
`loadConfig()`, which throws without `JELLYFIN_URL`/`JELLYFIN_API_KEY` set,
and (b) ran a startup IIFE that does a **live** `jellyfin.fetchUsers()` call
followed by `process.exit(1)` on failure and an unconditional `app.listen(...)`
— so a static `import` of anything from `server.ts` was previously
impossible to do safely inside a test process (it would either throw or,
worse, call `process.exit(1)` and kill the whole test run — reproduced this
exact failure mode mid-session before adding the `isEntryPoint` guard). The
fix: (1) extract `createApp(config, jellyfin, appState)` so route wiring is
callable with injected fakes; (2) gate the pre-existing startup IIFE behind
`require.main === module`, which is `true` for every real production
invocation (`npm start`/`dev:server`/Docker `CMD` all run `server.ts`/
`server.js` directly) and only `false` when a test `import()`s the module —
so production startup order, the live Jellyfin fetch, and `app.listen` are
completely unchanged for every existing invocation path. Confirmed via
`git diff --ignore-all-space -- src/server/server.ts`: the entire ~800-line
route table has zero semantic diff, only the two additive wrapper/guard
insertions.

The test still needs `JELLYFIN_URL`/`JELLYFIN_API_KEY` set (for
`loadConfig()` to not throw on import) — stubbed with dummy values in
`test.before()` before a dynamic `import('./server.js')` (a static import
would hoist above the env-stubbing statement; this file's module system,
matching the rest of `src/server`, is CommonJS-targeted so top-level `await`
isn't available either, hence the `test.before` hook rather than top-level
await). No live Jellyfin reachability is ever exercised: every test pre-seeds
`appState` with an already-existing party for the fixed Alice/Bob member set,
so `handleCreateParty`'s only Jellyfin-calling branch (`!exists` →
`jellyfin.ensurePartyUser`) is never taken.

No disagreement between any `/api/group*` alias and its `/api/party*`
counterpart was found — all five pairs (create, verify-pins, list w/
`groupKey`/`partyKey` per-entry equality, clear, session dual-alias) produced
byte-identical response bodies in every test run.
